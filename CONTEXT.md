# Yindee Frame Customizer: Project Context

เอกสารนี้ระบุบริบทของโปรเจกต์ โครงสร้างไฟล์ สถาปัตยกรรมระบบ และแนวทางการพัฒนาต่อ เพื่อให้ผู้พัฒนาหรือ AI Coding Agent ท่านอื่นสามารถทำงานต่อได้อย่างมีประสิทธิภาพ

---

## 🎯 1. วัตถุประสงค์ของโปรเจกต์ (Project Goal)
แอปพลิเคชันสำหรับให้ลูกค้าของร้าน **Yindee Frame** เข้ามาปรับแต่งภาพถ่ายและกรอบรูปขนาด **4x6 นิ้ว** (อัตราส่วน 2:3 แนวตั้ง หรือ 3:2 แนวนอน) ได้ด้วยตัวเอง โดยมี 2 โหมดการใช้งานหลัก:
1. **โหมดออกแบบเอง (Custom Design):** ลูกค้าเลือกสีกรอบ (ขาว, ดำ, ไม้โอ๊ค), เลือกสี Mat Board (กระดาษรองการ์ดด้านใน), เลือกแพทเทิร์นช่องใส่รูป (1-3 รูป) และอัปโหลดภาพมาปรับตำแหน่งเองได้อิสระ
2. **โหมดเทมเพลตสำเร็จรูป (Preset Template):** ลูกค้าเลือกจากชุดลวดลายสำเร็จรูปที่เจาะรูโปร่งใสเป็นเวกเตอร์ (.svg) และอัปโหลดรูปภาพใส่เข้าไปใต้เทมเพลตได้อย่างรวดเร็ว

ผลลัพธ์สุดท้ายคือ ลูกค้าสามารถดาวน์โหลดภาพดีไซน์ความละเอียดสูงระดับสั่งพิมพ์ได้ทันที และระบบจะช่วยอำนวยความสะดวกในการกดแชร์/ส่งรูปและรายละเอียดสเปกข้ามไปยังแอปพลิเคชัน **LINE OA** ของร้านค้าเพื่อสั่งตัดจริง

---

## 💻 2. เทคโนโลยีที่ใช้ (Tech Stack)
* **Framework:** Next.js (App Router, React, TypeScript)
* **Styling:** Tailwind CSS v4 (พร้อม PostCSS เคลียร์สไตล์ระบบแบบ HSL mapping)
* **Icons:** Lucide React
* **Graphics/Canvas:** Native HTML5 Canvas 2D API (สำหรับการคำนวณเวกเตอร์พิกัด เลื่อน ซูม หมุน และคลิปรูปภาพ)

## 🛠️ 4. สถาปัตยกรรมและตรรกะสำคัญของระบบ (Core Architecture & Logic)

### A. ตรรกะรูปภาพบน Canvas (HTML5 Canvas Engine)
* **Image Caching (`useRef`):** เพื่อลดการกระตุก (Flickering) และการโหลดรูปซ้ำๆ ในทุกเสี้ยววินาทีเมื่อขยับสไลเดอร์ ระบบจะเก็บบลอบไฟล์ภาพที่ถูกแปลงเป็น `HTMLImageElement` ลงใน `imageCacheRef` เมื่อมีสเตตัสเปลี่ยนจะดึงภาพจากแคชมาวาดใหม่ทันที
* **Object-Fit Cover:** คำนวณขยายภาพต้นฉบับให้ครอบคลุมช่องรูป (Slot) ทั้งความกว้างและความสูงโดยอิงจากสัดส่วนจริงของรูปภาพ เพื่อไม่ให้เห็นขอบดำหรือขอบโปร่งใส:
  `const baseScale = Math.max(slotW / img.width, slotH / img.height);`
* **Rotation-Aligned Dragging:** แปลงพิกัดการวาดและเมาส์โดยสั่งเลื่อนแกนพิกัด (`ctx.translate`) ก่อนหมุนพิกัดการวาด (`ctx.rotate`) เสมอ ช่วยให้เวลาผู้ใช้ลากรูปภาพ ทิศทางการขยับบนหน้าจอจะเคลื่อนไหวตามเมาส์/นิ้วลากตรงแบบ 1:1 เสมอ ไม่ว่ารูปภาพนั้นจะหมุนอยู่กี่องศาก็ตาม

### B. การส่งออกภาพความละเอียดสูงชั่วคราว (Dual-Canvas Strategy)
* เพื่อความลื่นไหลระดับ 60 FPS บนมือถือ หน้าพรีวิวหลักจะรันบน Canvas ขนาดเบา (`800x1200px`)
* เมื่อลูกค้ากดดาวน์โหลดเพื่อส่งพิมพ์ ระบบจะทำการ **เมาท์ Canvas ความละเอียดสูงตัวที่สองขึ้นมาในโหมดซ่อน (`id="high-res-canvas"`)** ขนาด `2400x3600px` (หรือ 3600x2400px ในแนวนอน) เพื่อเรนเดอร์รายละเอียดทั้งหมดให้คมชัดสำหรับพิมพ์รูปจริง
* **Scale Factor:** ค่าองค์ประกอบต่างๆ (ความหนาเส้น, ความเบลอเงา, ขนาดฟอนต์) จะถูกคูณด้วย `scaleFactor = width / 800` เพื่อให้รูปภาพพรีวิวและรูปภาพที่เซฟออกมามีหน้าตาตรงกัน 100%
* เมื่อดาวน์โหลดไฟล์ภาพสำเร็จ ระบบจะทำการ **ปลดเมาท์ (Unmount)** Canvas ความละเอียดสูงออกทันทีเพื่อคืนหน่วยความจำเครื่อง (ป้องกันบราว์เซอร์บนมือถือแครช)

### C. การรองรับเบราว์เซอร์ในแอปพลิเคชัน LINE (In-App Browser & HTTPS Fallbacks)
* **Mobile Scroll Override:** ป้องกันการเลื่อนหน้าจอของบราวเซอร์มือถือขณะลูกค้าใช้นิ้วลากขยับรูปภาพในกรอบ ด้วยการจับอีเวนต์ `touchmove` แบบดิบที่มีคุณสมบัติ `{ passive: false }` และควบคุมคุณสมบัติ CSS `touch-action: none` ชั่วคราวเมื่อลากวาง
* **Secure Context Check:** ฟังก์ชันการคัดลอกรายละเอียดข้อมูลลงคลิปบอร์ดออโต้วิเคราะห์สิทธิ์ `navigator.clipboard` หากแอปเปิดในเว็บวิวที่ไม่ได้อยู่บนโปรโตคอล HTTPS (เช่น การเทสภายในเครื่องหรือผ่านเครือข่ายอินเทอร์เน็ตทั่วไป) ระบบจะสลับไปใช้ `document.execCommand('copy')` เพื่อรองรับการทำงานกับบราวเซอร์อินแอปของ LINE ได้อย่างไม่มีปัญหา
